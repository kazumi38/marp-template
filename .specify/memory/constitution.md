# marp-template 憲法

## コア原則

### スライドは上司向け説明資料として設計する
このリポジトリのスライドは上司が意思決定できるように設計する。目的は「短時間で状況を把握し、判断できること」であり、情報の冗長さを排除して視認性と論理構造を優先する。

### タイトルスライドのレイアウト基準
タイトルスライドには見出しを中央に28ptで配置し、その下に部署名と氏名を記載する。見出しと補助情報の位置関係を固定し、視線を自然に先導することを必須とする。

### 本文スライドの構造基準
本文スライドのタイトルは右上に18ptで置き、タイトル下に端から端までの3pt水平線を引く。水平線の下にリード文を16ptで置き、各セクションが一目でわかる構造を維持する。

### デザインパターンの多様性
本文スライドではカードレイアウト（2つ、3つ、4つ並び）、フローチャート、表など、少なくとも30種類のデザインパターンを準備する。パターンは実装可能で再利用可能な形で示し、比較検討を容易にする。

### カラースキームと強調ルール
全体の基本文字色は #232F3E とし、強調用の赤と青は原色ではなく75%程度に調整する。色は可読性を損なわないことを最優先とし、強調は視線誘導と情報階層化のためにのみ使用する。

## デザイン制約
- マルチプラットフォーム出力で再現できる Marp Markdown と CSS の組み合わせを採用する。
- 日本語表記の可読性を維持し、行間・余白・フォントサイズを一貫させる。
- テーマと内容を分離し、テンプレートは再利用可能な形式で管理する。
- すべてのスライドは `template.md` などに記録し、PDF/HTML の両方で確認できるようにする。

## 開発ワークフロー
- デザイン変更は PR で行い、スライド例と画面キャプチャを添えてレビュアーが確認できるようにする。
- すべての変更は本憲法の原則と整合している必要があり、違反する場合は理由を明示する。
- 主要なスタイルやレイアウト変更は `README.md` または `template.md` に記録する。
- レビュー時には必ず配色、フォントサイズ、レイアウト構造の遵守を確認する。

## ガバナンス
本憲法はこのリポジトリのスライド設計とドキュメント品質の基準を定める。変更は PR で提案し、目的と影響を明確に記述する。改正時には関連ドキュメントも更新し、レビューで本憲法への準拠を確認する。

- すべてのPRはデザイン原則の遵守をチェックする。
- 変更は説明可能でなければならない。
- 重大なレイアウト/配色変更はレビュー時に承認を得る。
- 進行中の設計方針はこの憲法に従って管理する。
