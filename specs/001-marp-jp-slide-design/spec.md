# Feature Specification: Marp日本語スライドデザインテンプレート

**Feature Branch**: `001-marp-jp-slide-design`  
**Created**: 2026-04-06  
**Status**: Draft  
**Input**: User description: "Marpで日本語のスライドを作成するためのデザインを作りたい。上司向け説明スライドのテンプレートとレイアウト仕様を作成する。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 上司向けタイトルおよび本文レイアウトの提供 (Priority: P1)

マルプレポートを作成する担当者として、上司に説明するスライドを短時間で用意したい。標準化されたタイトルスライドと本文スライドの基本レイアウトがあれば、構成作業をすばやく始められる。

**Why this priority**: 上司向け資料の第一印象と構造が最も重要であり、まず基本テンプレートの品質を確保する必要があるため。

**Independent Test**: テンプレートを開き、タイトルスライドと本文スライドのレイアウトが要件どおり配置されていることを確認する。

**Acceptance Scenarios**:

1. **Given** 新しいスライドテンプレートが開かれている、 **When** タイトルスライドを表示すると、 **Then** タイトルは中央に28ptで表示され、その下に部署名と氏名が表示される。
2. **Given** 本文スライドが表示されている、 **When** 本文タイトルを確認すると、 **Then** タイトルは右上に18ptで表示され、下に端から端までの3pt水平線が引かれ、その下に16ptのリード文がある。

---

### User Story 2 - 多様な本文デザインパターンの準備 (Priority: P2)

資料作成者として、本文スライドで使える複数のデザインパターンを選びたい。デザインパターンが整理されていれば、伝えたい内容に応じて最適なレイアウトをすぐに選択できる。

**Why this priority**: 30種類のパターンは資料の表現力と再利用性を高めるため、テンプレートの差別化に直結する。

**Independent Test**: 用意された本文パターンを数え、カードレイアウト・フローチャート・表を含むことを確認する。

**Acceptance Scenarios**:

1. **Given** 本文スライドデザインの一覧が表示されている、 **When** カードレイアウトパターンを確認すると、 **Then** 2列、3列、4列のカード表示パターンが含まれている。
2. **Given** 本文デザインが一覧にある、 **When** フローチャートおよび表のパターンを確認すると、 **Then** それぞれ1つ以上のパターンが含まれている。

---

### User Story 3 - 色の強調ルールと可読性の確保 (Priority: P3)

資料作成者として、全体の文字色と強調色が定まっていると、スライドの統一感を保ちながら強調を適切に行える。

**Why this priority**: 色の統一は上司向け資料の信頼性と読みやすさに影響するため、パターン設計後の整合性確認として必要。

**Independent Test**: テンプレート中の文字色と強調色が指定のカラーコードで定義され、原色ではない75%相当の赤と青が利用可能であることを確認する。

**Acceptance Scenarios**:

1. **Given** テーマカラーの定義ファイルがある、 **When** 文字色を確認すると、 **Then** 基本文字色 #232F3E が適用されている。
2. **Given** 強調用の色が表示されている、 **When** 赤と青を確認すると、 **Then** どちらも原色ではなく75%程度に調整された色である。

---

### Edge Cases

- 内容が長くて1スライドに収まらない場合、カードレイアウトや表のパターンが複数スライドにまたがっても構造を維持できること。
- タイトルまたはリード文が日本語フォントで長くなった場合でも、指定のフォントサイズと配置を守りつつ読みやすさを保つこと。
- 強調色が視認性を損なう場合には、背景とのコントラストを再確認すること。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: タイトルスライドテンプレートを含み、中央の28ptタイトルとその下に部署名・氏名を表示する。
- **FR-002**: 本文スライドテンプレートを含み、右上に18ptタイトルを配置し、その下に端から端までの3pt水平線を引き、水平線の下に16ptリード文を配置する。
- **FR-003**: 本文スライド用に少なくとも30種類のデザインパターンを提供し、カード表示、フローチャート、表を含むこと。
- **FR-004**: 全体の基本文字色を #232F3E に設定し、赤と青の強調色を原色ではなく75%相当で提供する。
- **FR-005**: テンプレートは日本語の上司向け説明資料として視認性と論理構造を優先し、再利用可能な形式で整理されている。

### Key Entities *(include if feature involves data)*

- **スライドテンプレート**: タイトルスライドと本文スライドのレイアウト設計を含む基本要素。
- **デザインパターンセット**: 30種類以上の本文レイアウトパターン、カードレイアウト、フローチャート、表を含むパターン集。
- **カラーパレット**: 基本文字色 #232F3E と、75%調整の赤・青強調色。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 上司向け資料のタイトルスライドと本文スライドを、テンプレートを使って5分以内に構成できる。
- **SC-002**: 提供された本文デザインパターンが30種類以上あり、レビューで確認できる。
- **SC-003**: 全体の文字色として #232F3E が一貫して使用され、赤と青の強調色が75%相当で定義されていること。
- **SC-004**: レビュー時に、上司向け説明用資料の構造と可読性が「適切」と評価される。

## Assumptions

- この機能は Marp を用いた日本語スライドのデザインテンプレートに限定され、スライドの実際のコンテンツ生成は含まない。
- 目的は上司向け説明資料の構造と視認性を改善することであり、追加のインタラクティブ機能やアニメーションは本フェーズの対象外。
- テンプレートは Marp の標準出力形式で再現できる想定とする。
- 対象ユーザーは日本語を理解する担当者であり、社内上司への報告資料を前提とする。
